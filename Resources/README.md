## 🎓 Certification Roadmaps

### Beginner Path
1. **eJPT (eLearnSecurity Junior Penetration Tester)**
   - Hands-on, practical exam
   - Great first cert for pentesting
   - [INE eJPT Course](https://ine.com/learning/certifications-local/ejpt-certification)

2. **CompTIA Security+**
   - Foundational security concepts
   - Required for many government jobs (DoD 8570)
   - [CompTIA Security+](https://www.comptia.org/certifications/security)

### Intermediate Path
3. **OSCP (Offensive Security Certified Professional)**
   - Industry gold standard for pentesting
   - 24-hour practical exam
   - [Offensive Security OSCP](https://www.offensive-security.com/pwk-oscp/)

4. **PNPT (Practical Network Penetration Tester)**
   - Modern alternative to OSCP
   - Includes OSINT and AD exploitation
   - [TCM Security PNPT](https://tcm-sec.com/pnpt/)

### Advanced Path
5. **OSWE (Offensive Security Web Expert)**
   - Advanced web application pentesting
   - White-box testing focus
   - [Offensive Security OSWE](https://www.offensive-security.com/awae-oswe/)

6. **OSEP (Offensive Security Experienced Penetration Tester)**
   - Evasion techniques, advanced AD attacks
   - [Offensive Security OSEP](https://www.offensive-security.com/pen300-osep/)

7. **OSCE3 (Offensive Security Certified Expert 3)**
   - Combination of OSEP, OSED, OSWE
   - Highest level of Offensive Security certs
   - [Offensive Security OSCE3](https://www.offensive-security.com/offsec-pro-certifications/)

## 🏆 Bug Bounty Platforms & Tips

### Platforms
- **[HackerOne](https://hackerone.com/)** — Largest platform, top-tier programs (Uber, Twitter, etc.)
- **[Bugcrowd](https://bugcrowd.com/)** — Diverse programs, strong community.
- **[Intigriti](https://intigriti.com/)** — European-focused, growing fast.
- **[YesWeHack](https://yeswehack.com/)** — Major European platform.
- **[HackenProof](https://hackenproof.com/)** — Crypto/Web3 focused.

### Tips for Success
1. **Start with VDPs** (Vulnerability Disclosure Programs) — No money, but easier to find bugs.
2. **Niche Down** — Focus on one bug class (e.g., IDOR, SSRF, XSS) and master it.
3. **Read Disclosed Reports** — Learn from others on HackerOne Hacktivity.
4. **Automate Recon** — Use tools like `subfinder`, `httpx`, `nuclei`.
5. **Write Good Reports** — Clear title, impact, steps to reproduce, PoC.

## 🛠️ Essential Toolkits

### Reconnaissance
- **[Amass](https://github.com/owasp-amass/amass)** — Deep subdomain enumeration.
- **[Nmap](https://nmap.org/)** — Network scanning standard.
- **[Shodan](https://www.shodan.io/)** — Search engine for IoT and exposed services.

### Web Testing
- **[Burp Suite](https://portswigger.net/burp)** — The definitive web testing proxy.
- **[OWASP ZAP](https://www.zaproxy.org/)** — Free, open-source alternative to Burp.
- **[Nuclei](https://nuclei.projectdiscovery.io/)** — Fast, template-based vulnerability scanner.

### Active Directory
- **[BloodHound](https://github.com/BloodHoundAD/BloodHound)** — Visualize AD attack paths.
- **[CrackMapExec](https://github.com/Porchetta-Industries/CrackMapExec)** — Swiss-army knife for AD.
- **[Impacket](https://github.com/SecureAuthCorp/impacket)** — Python library for network protocols.

### C2 Frameworks
- **[Sliver](https://github.com/BishopFox/sliver)** — Open-source, cross-platform C2.
- **[Mythic](https://github.com/its-a-feature/Mythic)** — Modular, collaborative C2.
- **[Havoc](https://github.com/HavocFramework/Havoc)** — Modern, post-exploitation framework.
